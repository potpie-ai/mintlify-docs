**Ask**, **Build**, and **Debug** each operate on the same <a href="/concepts/knowledge-graph" className="mode-link">knowledge graph</a>, a structured map of every component, dependency, and relationship across the codebase.

| Mode | Use when |
|---|---|
| <a href="/build-flow/ask-a-question" className="mode-link">Ask</a> | Use when a question about the codebase needs an answer, whether that is how a flow operates, where something is implemented, or what depends on a given component |
| <a href="/build-flow/build-a-feature" className="mode-link">Build</a> | A feature, endpoint, or modification needs to be implemented with full awareness of existing patterns and dependencies |
| <a href="/build-flow/debug-an-issue" className="mode-link">Debug</a> | A reported issue, exception, or unexpected behavior needs to be traced from where it surfaces to where it originates |

## How they fit together

**Ask**, **Build**, and **Debug** operate on the same <a href="/concepts/knowledge-graph" className="mode-link">knowledge graph</a>. Using **Ask** first surfaces the relevant components and their relationships, narrowing what **Build** needs to generate or what **Debug** needs to trace.
